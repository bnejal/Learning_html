Adding Images

When inserting a picture into web you'll have to describe the image's location through the "src" tag, an alternate text description if the image doesn't load through the "alt" tag and a "title tag" to provide a description of the photo when the user hovers their cursor over the picture.

Height and width

One of the useful things of describing the image to the browser firsthand is that the browser knows how much space to eave a picture while a page is still loading(as pictures take a longer time to load than textual descriptors a.k.a. text tags) height can be described through the "height" attribute and width can be described through the "width" attributes

Three rules for creating images

there are three rules for creating images for your website which are summarized below

  1. Save images in the right format
    Webpages normally load jpeg, gid or png format. If you use the wrong format your webpage may load slowly, look not as sharp or not load at all. As all these formats have unique features that are application or use specific.
  2. Save images at the right size
    You should save the image at the same width and height it will appear on the website(measured in pixels). If the image is smaller than the width or height that you have specified , the image can be stretched and distorted. If the image is larger than the size you specified it may take longer for the image to be loaded.
  3. Measure Images in Pixels
    Computer screens are made up of tiny squares known as pixels. The number of pixels shown per inch of screen can vary if the user increases or decreases the resolution. Therefore when you are saving images at the right size you should always beasure them by pixel.

Image Formats

  1. JPEG
    JPEGs are extremely useful for images of many subtle shades of color or high detail. Like photos of fog or photos snow
  3. GIF/Png
    Gifs and pngs are good for photos composed of few colors and little color variation. Often filled with nothing but flat colors
  4. Animated Gifs
    Animations stored as gifs, often relatively simple/compact they served as boot screens for websites or played small clips
  5. SVG (Scalable Vector Graphics)
    Rescalable images incapable of losing fidelity due to the algorithmic generation of their objects
  6. Transparency
    Gifs for 100% transparency in an image. PNGs for drop shadows, rounded edges or semi-opaqness(basically use this format if you want variety with transparencies)
